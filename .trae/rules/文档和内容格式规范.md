# 文档和内容格式规范

## 1. 文档格式铁律

### 1.1 格式规则

- 标题层级严格对应 Markdown 层级
- 编号连续、不跳级、不重复、不乱序
- 段落之间空一行，重点内容加粗
- 英文术语首次出现必须标注中英文全称
- 步骤用有序列表，知识点用无序列表，对比用表格
- 使用 RsPress 内置组件：`:::tip` / `:::warning`
- 所有文档必须使用 包含 **Frontmatter 的 Markdown**  格式（.md）。仅在需要自定义组件时使用 MDX
- 禁止：非标准 Markdown 语法、混乱嵌套

### 1.2 强制标题格式

必须区分文档内容格式（本文规则）和 docs 下文件夹格式（RsPress 构建规则）。

- 一级标题：`# XXXX`
- 二级标题：`## 1. XXXX`
- 三级标题：`### 1.1 XXXX`
- 衍生标题以此类推。

### 1.3 文档结构示例

```
---
title: 文档标题
description: 文档简短描述
author: LC
date: YYYY-MM-DD
category: 分类
---

# 一级标题

## 1. 二级标题（按序递增）

### 1.1 三级标题（按序递增）

### 1.2 三级标题（按序递增）

## 2. 二级标题（按序递增）

```

## 2. 图片下载处理规则

### 2.1 下载能力判断

| 能力 | 说明 |
|-----|------|
| ✅ 能下载 | 直接可访问的URL（无需登录、无付费墙） |
| ❌ 不能下载 | URL无效、需要登录权限、网站限制反爬 |

### 2.2 处理流程

**情况1：能成功下载**
- 使用 `scripts/download-images.py` 脚本辅助下载（参见脚本文档）
- 下载到本地 `public/images/{章节名}/`（按文档所属章节分类，无章节则默认 `default` 文件夹）
- 使用描述性文件名，避免随机名，如 `agent-architecture-overview.png`
- 自动更新 `image_manifest.csv` 中的引用路径
- 支持 所有常见图片格式：`.png`, `.jpg`, `.jpeg`, `.gif`, `.svg`, `.webp` 等

**情况2：下载失败**
- 在文档对应位置标注 `:::warning 找不到原始图像`
- 注明原始引用地址和出处信息
- 输出失败列表供用户手动处理

**情况3：用户提供新图片**
- 接收用户提供的有效图片URL或本地路径
- 同**情况1**处理流程

## 3. 知识内容规范

### 3.1 内容规范

- 术语统一：中英文全称+缩写（首次出现必须标注，如 LLM (Large Language Model)）
- 易混淆概念必须用表格对比
- 严禁：编造数据、虚构实践、非权威结论
