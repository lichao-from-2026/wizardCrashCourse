# 文档和内容格式规范

## 1. 文档格式铁律

### 1.1 强制标题格式

必须区分文档内容格式（本文规则）和 docs 下文件夹格式（RsPress 构建规则）。

- 一级标题：`# 一、XXXX`
- 二级标题：`## 1. XXXX`
- 三级标题：`### 1.1 XXXX`
- 四级标题：`#### 1.1.1 XXXX`

### 1.2 格式规则

- 标题层级严格对应 Markdown 层级
- 编号连续、不跳级、不重复、不乱序
- 段落之间空一行，重点内容加粗
- 英文术语首次出现必须标注中英文全称
- 步骤用有序列表，知识点用无序列表，对比用表格
- 使用 RsPress 内置组件：`:::tip` / `:::warning`
- 所有文档必须使用 包含 **Frontmatter 的 Markdown**  格式（.md），仅在需要自定义组件时使用 MDX
- 禁止：复杂内嵌 HTML、非标准 Markdown 语法、混乱嵌套

## 2. 知识内容规范

### 2.1 内容规范

- 术语统一：中英文全称+缩写（首次出现必须标注，如 LLM (Large Language Model)）
- 易混淆概念必须用表格对比
- 严禁：编造数据、虚构实践、非权威结论

### 2.2 文档结构示例

```
---
title: 文档标题
description: 文档简短描述
author: 作者名
date: YYYY-MM-DD
category: 分类
---

# 一、标题

## 1. 二级标题

```
