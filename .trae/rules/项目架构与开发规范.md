# AI Agent 知识库项目 - 开发规范

## 目录结构

```
src/
├── components/    # 组件目录，仅负责渲染
│   └── [Name]/   # 每个组件独立文件夹
│       ├── [Name].tsx                  # 组件主文件（仅渲染逻辑）
│       ├── [Name].module.css  # CSS Modules 样式
│       ├── types.ts     # 类型定义
│       ├── hooks.ts     # 状态管理
│       └── index.ts     # 统一导出
├── hooks/         # 全局 Hooks（跨组件）
├── utils/         # 纯函数工具（无副作用）
├── constants/     # 全局常量
└── types/         # 类型定义（含 env.d.ts）
```

## 核心原则

### 1. 组件规范
- **index.tsx**：仅包含导入和 JSX 渲染
- **hooks.ts**：所有状态管理和业务逻辑
- **导入无扩展名**：`./ComponentName` 而非 `./ComponentName.tsx`
- **组件名**：PascalCase，如 `DocStatusTable`

### 2. 注释规范
- **所有文件必须加中文注释**
- **文件头**：`/** */` 格式说明用途
- **关键逻辑**：`//` 单行注释解释用途
- **重要属性**：必须有注释说明

### 3. 样式规范
- 使用 CSS Modules：`[Name].module.css`
- 类名 camelCase：`statusItem`
- 充分中文注释每个样式块

### 4. TypeScript 配置
```json
{
  "include": ["src/**/*", "rspress.config.ts"],
  "strict": true
}
```
- 必须包含 `src/**/*`，否则 env.d.ts 不生效
- `src/types/env.d.ts`：声明 CSS Modules、图片等资源类型

## 开发清单

创建新组件时：
- [ ] 目录结构完整（types、hooks、constants、css、index）
- [ ] 组件逻辑在 hooks 中，仅渲染在 tsx
- [ ] 充分中文注释
- [ ] 运行 build 验证
