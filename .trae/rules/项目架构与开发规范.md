# RsPress 知识库项目 - 开发规范

## 目录结构

```
[知识库项目名]/
├── docs/                    # 知识库文档（RsPress 主内容）
│   ├── index.mdx            # 知识库首页（不加入导航配置，默认访问跳转）
│   ├── 01-[章节名]/         # 按章节分类，序号递增
│   │   ├── 01-[知识点]/     # 知识点有多篇文档时创建文件夹
│   │   │   ├── 01-[文档名].md
│   │   │   └── 02-[文档名].md
│   │   ├── 02-[知识点]/
│   │   │   └── 01-[文档名].md
│   │   └── 03-[单篇文档].md # 仅有一篇文档时直接放 md（无需创建文件夹）
│   ├── 02-[章节名]/
│   └── _nav.json            # 全局侧边栏导航配置
│
├── public/                  # 静态资源目录
│   └── images/              # 图片资源
│       ├── 01-[章节名]/     # 按章节分类存放，与 docs/ 对应
│       ├── 02-[章节名]/
│       └── default/         # 默认分类
│
├── scripts/                 # 辅助脚本目录
│   └── download-images.py   # 图片下载脚本
│
├── src/                     # RsPress 自定义组件/扩展（如需要）
│   ├── components/          # 自定义组件
│   │   ├── [Name]/          # 每个组件独立文件夹（PascalCase）
│   │   │   ├── [Name].tsx          # 组件主文件（仅渲染逻辑）
│   │   │   ├── [Name].module.css   # CSS Modules 样式
│   │   │   ├── hooks.ts             # 状态管理和业务逻辑
│   │   │   └── index.ts             # 统一导出（组件、Hook）
│   │   └── index.ts          # 组件目录总导出
│   ├── hooks/               # 全局 Hooks
│   ├── utils/               # 工具函数
│   ├── constants/           # 全局常量（所有常量分类存放）
│   └── types/               # 全局类型定义（所有类型存放）
│       └── env.d.ts
│
├── .trae/                   # Trae 配置和规则
│   └── rules/               # 项目规则文档
│
├── AGENT.md                 # Agent 配置和使用说明
├── rspress.config.ts        # RsPress 配置文件
├── package.json             # 依赖配置
└── README.md                # 项目说明
```

## 核心原则

### 1. 组件规范
- **[Name].tsx**: 仅包含导入和 JSX 渲染
- **hooks.ts**: 所有状态管理和业务逻辑
- **index.ts**: 统一导出（组件、Hook）
- **导入无扩展名**: `./ComponentName` 而非 `./ComponentName.tsx`
- **组件名**: PascalCase，如 `DocStatusTable`
- **导出示例**:
  ```typescript
  export { ComponentName, default } from './ComponentName';
  export type { ComponentNameProps, TypeName } from '../../types/xxx'; // 从全局类型导入
  export { useComponentName } from './hooks';
  ```

### 2. 常量规范
- **全局常量**: 所有常量都放在 `src/constants/` 目录下分类存放
- **常量文件名**: 描述性名称，如 `doc-status.tsx`、`status-selector.ts`
- **常量导出**: 在 `src/constants/index.ts` 统一导出，避免命名冲突
- **选择文件扩展名**: 含 JSX 用 `.tsx`，纯 TypeScript 用 `.ts`

### 3. 类型规范
- **全局类型**: 所有类型都放在 `src/types/` 目录下分类存放
- **类型文件名**: 描述性名称，如 `doc-status.ts`
- **类型导出**: 在 `src/types/index.ts` 统一导出
- **共享类型**: 组件间共享的类型必须放在全局类型目录

### 4. 注释规范
- **所有文件必须加中文注释**
- **文件头**：`/** */` 格式说明用途、作者、日期
- **关键逻辑**：`//` 单行注释解释用途
- **重要属性**：必须有注释说明

### 5. 样式规范
- 使用 CSS Modules：`[Name].module.css`
- 类名 camelCase：`statusItem`
- 充分中文注释每个样式块

### 6. TypeScript 配置
```json
{
  "include": ["src/**/*", "rspress.config.ts"],
  "strict": true
}
```
- 必须包含 `src/**/*`，否则 env.d.ts 不生效
- `src/types/env.d.ts`：声明 CSS Modules、图片等资源类型

## 开发清单

创建新组件时：
- [ ] 目录结构完整（[Name].tsx、[Name].module.css、hooks.ts、index.ts）
- [ ] 组件逻辑在 hooks 中，仅渲染在 tsx
- [ ] 常量放在 `src/constants/` 对应文件中
- [ ] 类型放在 `src/types/` 对应文件中
- [ ] index.ts 完整导出（组件、Hook）
- [ ] 充分中文注释
- [ ] 运行 build 验证
