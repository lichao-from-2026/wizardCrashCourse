# 一、学习路径大纲

本大纲重点突出提示词工程（Prompt Engineering）、上下文工程（Context Engineering）、Harness 工程三大核心工程，详细说明在项目中如何与 AI Agent 配合、最佳实践及注意事项。

## 1. 前置知识：AI 相关概念术语（1 周）

### 目标
建立 AI 基础概念体系，为 Agent 学习打下扎实基础。

### 学习内容
1. 基础概念（入门级）
   - 人工智能（Artificial Intelligence）：模拟人类智能的计算机系统
   - 机器学习（Machine Learning）：让计算机从数据中学习的方法
   - 深度学习（Deep Learning）：使用神经网络的机器学习方法
   - 神经网络（Neural Network）：模仿人脑结构的计算模型
   - 自然语言处理（Natural Language Processing）：让计算机理解和处理人类语言

2. 大语言模型（Large Language Model）基础
   - Transformer：现代 LLM 的基础架构
   - 预训练（Pretraining）：在大量文本上训练模型
   - 微调（Fine-tuning）：在特定任务上调整模型
   - 上下文窗口（Context Window）：模型能处理的文本长度
   - Token：LLM 处理的基本单位

3. 核心技术（进阶）
   - 向量嵌入（Vector Embedding）：文本的数值表示
   - 向量数据库（Vector Database）：存储和检索向量的数据库
   - 检索增强生成（Retrieval-Augmented Generation）：结合检索和生成的技术
   - 提示词工程（Prompt Engineering）：设计有效的提示词
   - 函数调用（Function Calling）：模型调用外部函数的能力

4. Agent 相关概念
   - 智能体（Agent）：能自主执行任务的系统
   - 链（Chain）：将多个步骤组合的序列
   - 工具（Tool）：Agent 可以使用的外部功能
   - 记忆（Memory）：Agent 存储信息的能力
   - 规划（Planning）：Agent 制定执行计划的能力

### 实践任务
- 查阅并整理 10 个核心 AI 概念的定义和应用场景
- 完成一个简单的 LLM API 调用示例
- 尝试使用不同的提示词模式，观察模型输出的差异

## 3. 阶段二：提示词工程（Prompt Engineering）（2 周）—— 核心重点

### 目标
掌握提示词工程的核心方法和最佳实践，能够设计高质量、可复用的提示词。

### 学习内容
1. 提示词基础
   - 提示词的作用与重要性
   - 提示词的基本结构：指令、上下文、输入、输出格式
   - 提示词的设计原则

2. 核心提示词技巧
   - 清晰明确的指令
   - 提供上下文信息
   - 指定输出格式
   - 分步骤引导

3. Few-shot 学习
   - 什么是 Few-shot learning
   - 示例的设计原则
   - 示例数量对效果的影响
   - 示例的多样性

4. Chain-of-Thought 提示
   - 思维链的概念
   - 何时使用思维链
   - 思维链提示的设计
   - 思维链的变体（Zero-shot CoT）

5. ReAct 模式
   - ReAct 的概念：Reasoning + Acting
   - ReAct 提示词的结构
   - 思考、行动、观察的循环
   - ReAct 的实际应用

6. 提示词模式汇总
   - Zero-shot 提示
   - Few-shot 提示
   - Chain-of-Thought 提示
   - ReAct 提示
   - 自我一致性（Self-Consistency）
   - Tree-of-Thought

7. 提示词工程最佳实践
   - 迭代优化提示词
   - A/B 测试提示词
   - 提示词版本管理
   - 提示词模板化
   - 提示词的可移植性

8. 与 AI Agent 配合的提示词设计
   - 角色定义提示词
   - 任务分解提示词
   - 工具调用提示词
   - 反思与优化提示词

9. 注意事项
   - 避免模糊不清的表述
   - 注意提示词长度限制
   - 防止提示词注入攻击
   - 考虑模型偏见

### 实践任务
- 为 5 个不同任务设计提示词模板
- 实现一个 Chain-of-Thought 提示词
- 编写一个完整的 ReAct Prompt
- 对比不同提示词模式的效果差异
- 建立个人的提示词模板库

## 4. 阶段三：上下文工程（Context Engineering）（2 周）—— 核心重点

### 目标
掌握上下文管理的技术，有效利用 LLM 的上下文窗口，构建高效的 RAG 系统。

### 学习内容
1. 上下文窗口管理
   - 理解上下文窗口的限制
   - Token 计数与估算
   - 上下文窗口的最佳利用策略
   - 成本与效果权衡

2. 上下文构建策略
   - 信息优先级排序
   - 重要信息前置
   - 冗余信息剔除
   - 结构化上下文

3. RAG（检索增强生成）详解
   - RAG 的核心概念
   - RAG 的工作流程：索引 → 检索 → 生成
   - 向量数据库选型与对比
   - 分块策略（Chunking）
   - 嵌入模型选择

4. 检索优化
   - 相似度计算方法
   - 重排序（Reranking）
   - 混合检索（Hybrid Search）
   - 检索阈值设置
   - 检索结果数量优化

5. 上下文压缩与优化
   - 抽取式摘要
   - 抽象式摘要
   - 关键信息提取
   - 去重与合并
   - 长文档处理策略

6. 长上下文处理
   - Map-Reduce 模式
   - Refine 模式
   - 分层摘要
   - 滑动窗口

7. 与 AI Agent 配合的上下文工程
   - 对话历史管理
   - 工作记忆维护
   - 检索结果格式化
   - 上下文注入时机

8. 最佳实践
   - 分块大小的选择
   - 嵌入维度的权衡
   - 检索质量评估
   - 上下文更新策略

9. 注意事项
   - 上下文窗口溢出风险
   - 检索结果相关性
   - 信息完整性与简洁性平衡
   - 隐私与安全

### 实践任务
- 构建一个简单的 RAG 系统
- 尝试不同的分块策略并对比效果
- 实现上下文压缩功能
- 处理长文档问答场景
- 优化 RAG 系统的检索准确率

   - 转换链（Transformation Chain）

3. 状态管理与持久化

   - Prompt 调试技巧
   - 性能分析
   - 错误追踪

5. 测试与评估
   - 单元测试
   - 集成测试
   - 端到端评估
   - A/B 测试框架
   - 回归测试
   - 自动化测试

6. 与 AI Agent 配合的 Harness 工程
   - Agent 生命周期管理
   - 任务队列与调度
   - 资源隔离与限流
   - 错误重试与降级

7. 最佳实践
   - 代码组织规范
   - 配置管理
   - 部署策略
   - 监控告警
   - 容量规划

8. 注意事项
   - 避免过度工程化
   - 注意成本控制
   - 确保可扩展性
   - 关注安全性

### 实践任务

## 10. 在当前项目中与 AI Agent 配合的最佳实践

### 10.1 提示词工程实践
- **使用结构化提示词模板**：在项目中创建统一的提示词模板库
- **版本管理**：对重要提示词进行版本控制，记录变更历史
- **A/B 测试**：在生产环境中对比不同提示词的效果
- **提示词审查**：建立提示词审查机制，确保质量和安全

### 10.2 上下文工程实践
- **合理分块**：根据文档类型选择合适的分块大小
- **检索优化**：定期评估检索质量，优化检索策略
- **上下文更新**：建立上下文缓存和更新机制
- **隐私保护**：对敏感信息进行脱敏处理

### 10.3 Harness 工程实践
- **模块化设计**：遵循项目架构规范，组件化开发
- **可观测性**：集成 LangSmith 等工具进行追踪
- **自动化测试**：为 Agent 组件编写充分的测试用例
- **配置管理**：使用配置文件管理环境差异

### 10.4 协作流程
- **任务分解**：使用 Agent 协助分解复杂任务
- **代码审查**：利用 Agent 进行代码审查和优化建议
- **文档生成**：让 Agent 协助生成和维护文档
- **持续优化**：定期回顾 Agent 使用效果，迭代改进

### 10.5 注意事项
- **成本控制**：监控 LLM API 调用成本，避免浪费
- **质量保障**：重要输出需要人工审核
- **安全风险**：防止提示词注入、数据泄露等安全问题
- **性能优化**：合理使用缓存、异步等技术优化性能
