# 一、核心主题

本知识库以**提示词工程（Prompt Engineering）**、**上下文工程（Context Engineering）**、**Harness 工程**为三大核心重点，构建完整的 AI Agent 学习体系。

## 1. Agent 基础理论

- Agent 的定义与概念
- Agent 的核心组件
- Agent 的分类与架构模式
- 多 Agent 协作原理

## 2. Agent 技术栈 —— 三大核心工程

### 2.1 提示词工程（Prompt Engineering）—— 核心重点
- 提示词基础：结构、原则
- 核心技巧：清晰指令、上下文、输出格式
- Few-shot learning
- Chain-of-Thought 提示
- ReAct 模式
- 提示词模式汇总
- 最佳实践与注意事项
- 与 AI Agent 配合的提示词设计

### 2.2 上下文工程（Context Engineering）—— 核心重点
- 上下文窗口管理
- 上下文构建策略
- RAG（检索增强生成）详解
- 检索优化
- 上下文压缩与优化
- 长上下文处理
- 与 AI Agent 配合的上下文工程
- 最佳实践与注意事项

### 2.3 Harness 工程 —— 核心重点
- 模块化与组件化
- 管道编排
- 状态管理与持久化
- 可观测性与调试
- 测试与评估
- 与 AI Agent 配合的 Harness 工程
- 最佳实践与注意事项

### 2.4 其他技术栈组件
- LLM 作为 Agent 大脑
- 记忆系统设计
- 工具使用与调用
- 规划与推理能力
- 评价与优化方法

## 7. 学习路径设计原则

1. 从易到难：先理论后实践，先基础后进阶
2. 循序渐进：每阶段建立在前一阶段基础上
3. 理论实践结合：每个知识点都配有实践任务
4. 项目驱动：通过实际项目巩固所学知识
5. 重点突出：以三大工程为核心构建知识体系
6. 持续更新：跟随技术发展不断更新内容
