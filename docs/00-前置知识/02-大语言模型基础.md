# 一、大语言模型基础

**大语言模型（Large Language Model，简称 LLM）** 是 AI Agent 的核心大脑。本章将介绍 LLM 的核心概念、工作原理和关键技术。

## 1. 什么是大语言模型

大语言模型是一种基于深度学习的语言模型，它通过在海量文本数据上进行预训练，学习语言的统计规律和语义知识。

### 1.1 LLM 的关键特征

| 特征 | 描述 |
|------|------|
| **规模大** | 参数数量从数十亿到数万亿不等 |
| **预训练** | 在大规模通用文本数据上预训练 |
| **生成能力** | 能够生成连贯、有意义的文本 |
| **涌现能力** | 展现出一些预训练时未明确训练的能力 |

### 1.2 常见的大语言模型

| 模型 | 开发者 | 特点 |
|------|--------|------|
| GPT 系列 | OpenAI | 对话能力强，应用广泛 |
| Claude | Anthropic | 安全可靠，擅长长文本 |
| LLaMA | Meta | 开源，支持定制化 |
| Qwen | 阿里巴巴 | 多语言能力强 |
| Mistral | Mistral AI | 高效、性能优异 |

## 2. Transformer 架构

Transformer 是大语言模型的基础架构，由 Google 在 2017 年的论文《Attention Is All You Need》中提出。

### 2.1 Transformer 的核心组件

#### 2.1.1 自注意力机制（Self-Attention）

自注意力机制允许模型在处理每个词时，关注输入序列中的其他相关词。

**计算过程**：

1. 计算查询（Query）、键（Key）、值（Value）
2. 计算注意力分数
3. 应用 Softmax 得到注意力权重
4. 加权求和得到输出

**公式**：

```
Attention(Q, K, V) = softmax(QK^T/√d_k)V
```

#### 2.1.2 多头注意力（Multi-Head Attention）

多头注意力将自注意力机制扩展到多个"头"，每个头学习不同方面的注意力模式。

**优势**：
- 捕获不同类型的语义关系
- 提高模型的表达能力

#### 2.1.3 前馈神经网络（FFN）

每个 Transformer 层都包含一个前馈神经网络，通常是两个线性层，中间有一个激活函数（通常是 ReLU）。

#### 2.1.4 层归一化和残差连接

- **层归一化（Layer Normalization）**：稳定训练过程
- **残差连接（Residual Connection）**：缓解梯度消失问题

### 2.2 Transformer 结构

Transformer 包含两个主要部分：

1. **编码器（Encoder）**：处理输入序列
2. **解码器（Decoder）**：生成输出序列

**注意**：大多数现代 LLM（如 GPT 系列）只使用解码器部分，采用自回归方式生成文本。

## 3. Token 化

**Token（标记）** 是 LLM 处理文本的基本单位。文本首先被分割成 token 序列，然后被转换为数值表示。

### 3.1 常见的 Token 化方法

| 方法 | 描述 | 示例 |
|------|------|------|
| **词级 Token 化** | 按词分割 | "Hello world" → ["Hello", "world"] |
| **字符级 Token 化** | 按字符分割 | "Hello" → ["H", "e", "l", "l", "o"] |
| **子词 Token 化** | 按子词分割（最常用） | "unhappiness" → ["un", "happi", "ness"] |

### 3.2 Byte-Pair Encoding (BPE)

BPE 是最常用的子词 Token 化算法：

1. 初始词汇表包含所有单个字符
2. 迭代合并最常见的字符对
3. 直到达到预设的词汇表大小

### 3.3 Token 数量估算

不同语言和内容的 Token 数量不同：
- 英文：约 1000 Token = 750 词
- 中文：约 1000 Token = 500-600 汉字

## 4. 预训练

预训练是 LLM 训练的第一阶段，在大规模文本数据上进行自监督学习。

### 4.1 预训练目标

最常见的预训练目标是**下一个词预测（Next Token Prediction）**：

- 给定一段文本，预测下一个最可能出现的词
- 通过最大化真实词的概率来训练模型

### 4.2 预训练数据

预训练数据通常来自：
- 网页、书籍、文章
- 代码仓库
- 对话数据
- 其他公开文本资源

**数据要求**：
- 大规模（数万亿 Token）
- 多样性
- 质量高

### 4.3 预训练过程

1. **数据准备**：收集、清洗、过滤数据
2. **Token 化**：将文本转换为 token 序列
3. **模型训练**：使用大规模计算资源训练
4. **评估**：在测试集上评估模型性能

## 5. 微调

微调（Fine-tuning）是在预训练后，使用特定任务的数据进一步训练模型，使其适应特定任务。

### 5.1 微调的类型

#### 5.1.1 全量微调

- 更新模型的所有参数
- 需要大量计算资源
- 可能导致"灾难性遗忘"

#### 5.1.2 参数高效微调（Parameter-Efficient Fine-Tuning，PEFT）

只更新模型的一小部分参数：

- **LoRA（Low-Rank Adaptation）**：添加低秩矩阵
- **Adapter**：插入小型适配层
- **Prefix Tuning**：只优化前缀向量

### 5.2 指令微调（Instruction Tuning）

使用指令格式的数据微调模型，使其更好地遵循人类指令：

```
指令：请总结以下文章
输入：[文章内容]
输出：[总结内容]
```

### 5.3 人类反馈强化学习（RLHF）

通过人类反馈进一步优化模型：

1. 收集人类对模型输出的排序或评分
2. 训练奖励模型（Reward Model）
3. 使用强化学习（如 PPO）优化模型，使其输出更符合人类偏好

## 6. 上下文窗口

**上下文窗口（Context Window）** 是模型一次能处理的 token 数量限制。

### 6.1 常见的上下文窗口大小

| 模型 | 上下文窗口 |
|------|-----------|
| GPT-3.5 | 4K/16K |
| GPT-4 | 8K/32K/128K |
| Claude 3 | 200K |
| GPT-4o | 128K |

### 6.2 上下文窗口的影响

- **输入限制**：不能超过窗口大小
- **生成质量**：太长的上下文可能导致"中间部分遗忘"
- **成本**：通常与处理的 token 数量成正比

### 6.3 处理长上下文的方法

1. **截断**：只保留最重要的部分
2. **摘要**：对长文本进行摘要
3. **分块处理**：将长文本分成多个块处理
4. **检索增强**：只检索相关内容

## 7. LLM 的能力

### 7.1 基础能力

- **文本生成**：写作、对话、翻译
- **理解**：问答、分类、摘要
- **推理**：逻辑推理、数学计算
- **编程**：代码生成、调试

### 7.2 涌现能力

当模型规模足够大时，会出现一些预训练时未明确训练的能力：

- **链式思维推理**
- **多步推理**
- **复杂指令遵循**
- **工具使用**

### 7.3 局限

- **知识截止**：不知道训练数据之后的信息
- **幻觉**：可能生成虚假信息
- **一致性**：同一问题可能给出不同答案
- **推理**：复杂逻辑推理可能出错

## 8. 提示词工程的重要性

由于 LLM 通过预测下一个词来工作，提示词（Prompt）的设计直接影响输出质量。这就是为什么**提示词工程**是 AI Agent 的核心技术之一。

**好的提示词可以**：
- 提高输出准确性
- 控制输出格式
- 引导模型进行推理
- 减少幻觉

## 9. 总结

本章介绍了大语言模型的基础知识：

- **Transformer 架构**：LLM 的基础
- **Token 化**：文本处理的基本单位
- **预训练与微调**：模型训练的两个阶段
- **上下文窗口**：模型输入的限制
- **LLM 的能力与局限**

理解这些基础概念，将帮助我们更好地掌握后续的提示词工程、上下文工程等核心技术。
