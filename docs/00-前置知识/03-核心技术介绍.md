# 一、核心技术介绍

本章将介绍构建 AI Agent 所需的核心技术，包括向量嵌入、向量数据库、检索增强生成（RAG）、提示词工程等。

## 1. 向量嵌入

**向量嵌入（Vector Embedding）** 是将文本（或其他类型数据）转换为密集数值向量的技术。这些向量能够捕捉数据的语义信息。

### 1.1 什么是向量嵌入

向量嵌入将高维离散数据（如文本）映射到低维连续向量空间中。语义相似的文本在向量空间中距离更近。

**示例**：

```
"猫" → [0.2, -0.5, 0.8, ...] （256维、768维或更高维向量）
"狗" → [0.3, -0.4, 0.7, ...] （与"猫"的向量接近）
"汽车" → [-0.6, 0.3, -0.2, ...] （与"猫"的向量较远）
```

### 1.2 向量嵌入的特点

| 特点 | 描述 |
|------|------|
| **语义捕捉** | 向量间的距离表示语义相似性 |
| **维度固定** | 无论输入文本长度，输出向量维度相同 |
| **连续性** | 向量空间是连续的，便于计算 |
| **压缩性** | 相比原始文本，向量更紧凑 |

### 1.3 常见的嵌入模型

| 模型 | 开发者 | 特点 |
|------|--------|------|
| text-embedding-ada-002 | OpenAI | 通用性强 |
| text-embedding-3-small/large | OpenAI | 新的高性能嵌入 |
| sentence-transformers | 开源 | 多语言，开源免费 |
| e5 | Microsoft | 高性能检索 |
| bge | 智源 | 中文效果好 |

### 1.4 余弦相似度

计算两个向量之间的相似度常用**余弦相似度（Cosine Similarity）**：

```
cos(θ) = (A · B) / (||A|| ||B||)
```

其中：
- A · B 是向量点积
- ||A|| 和 ||B|| 是向量的范数

**范围**：[-1, 1]，值越大表示越相似

## 2. 向量数据库

**向量数据库（Vector Database）** 是专门用于存储和检索向量嵌入的数据库。

### 2.1 为什么需要向量数据库

传统数据库不适合高效处理向量检索：
- 向量维度高（几百到几千维）
- 需要计算相似度而不是精确匹配
- 数据量大时需要近似检索

### 2.2 常见的向量数据库

| 数据库 | 类型 | 特点 |
|--------|------|------|
| **Chroma** | 轻量级 | 易用，适合开发 |
| **Pinecone** | 云服务 | 托管服务，可扩展 |
| **Weaviate** | 开源 | 功能丰富，支持多种索引 |
| **FAISS** | 库 | Facebook 开发，高性能 |
| **Milvus** | 开源 | 企业级，可扩展 |
| **Qdrant** | 开源 | Rust 开发，高性能 |

### 2.3 向量检索的基本流程

1. **索引阶段**：
   - 将文档分块
   - 为每个块生成向量嵌入
   - 将向量存储在向量数据库中

2. **查询阶段**：
   - 将查询转换为向量
   - 在向量数据库中检索最相似的向量
   - 返回对应的文档块

### 2.4 近似最近邻算法

为了在大规模数据上高效检索，使用**近似最近邻（Approximate Nearest Neighbor，ANN）**算法：

| 算法 | 特点 |
|------|------|
| **HNSW** | 分层导航小世界图，精度高 |
| **IVF** | 倒排文件，检索快 |
| **PQ** | 乘积量化，节省空间 |

## 3. 检索增强生成

**检索增强生成（Retrieval-Augmented Generation，简称 RAG）** 是一种结合信息检索和文本生成的技术。

### 3.1 RAG 的动机

LLM 的局限性：
- **知识截止**：不知道训练数据之后的信息
- **幻觉**：可能生成虚假信息
- **知识更新难**：重新训练成本高

RAG 通过检索外部知识来解决这些问题。

### 3.2 RAG 的工作流程

```
用户提问
   ↓
查询向量化
   ↓
向量数据库检索
   ↓
获取相关文档
   ↓
构建提示词（查询 + 检索结果）
   ↓
LLM 生成回答
```

### 3.3 RAG 的优势

1. **知识时效性**：可以使用最新数据
2. **可溯源**：回答基于检索结果，可追溯
3. **减少幻觉**：基于真实数据生成
4. **灵活扩展**：可以添加新的知识源

### 3.4 RAG 的变体

| 类型 | 描述 |
|------|------|
| **Naive RAG** | 简单的检索 + 生成 |
| **Advanced RAG** | 包含预处理、重排序等优化 |
| **Modular RAG** | 模块化设计，灵活组合 |
| **Graph RAG** | 结合知识图谱 |

## 4. 提示词工程

**提示词工程（Prompt Engineering）** 是设计和优化提示词，以更好地引导 LLM 输出的技术。

### 4.1 提示词的基本结构

一个好的提示词通常包含：

1. **指令（Instruction）**：要求模型做什么
2. **上下文（Context）**：提供相关背景信息
3. **输入数据（Input Data）**：需要处理的具体数据
4. **输出格式（Output Format）**：指定输出的格式

**示例**：

```
指令：请总结以下文章
上下文：这是一篇关于人工智能的文章
输入：[文章内容]
输出格式：请用 3 个要点总结
```

### 4.2 核心提示词技巧

| 技巧 | 描述 |
|------|------|
| **清晰明确** | 避免模糊不清的表述 |
| **给出示例** | 使用 Few-shot 学习 |
| **指定格式** | 明确要求输出格式 |
| **分步骤** | 引导模型逐步推理 |
| **角色设定** | 给模型设定特定角色 |

### 4.3 常见的提示词模式

1. **Zero-shot**：直接给出任务，不提供示例
2. **Few-shot**：提供几个示例
3. **Chain-of-Thought**：引导模型逐步推理
4. **ReAct**：结合推理和行动

## 5. 函数调用

**函数调用（Function Calling）** 是 LLM 的一项重要功能，允许模型调用外部工具或函数。

### 5.1 函数调用的工作原理

1. **定义函数**：描述函数的名称、参数、功能
2. **模型决策**：LLM 决定是否需要调用函数
3. **执行函数**：开发者根据模型输出执行函数
4. **返回结果**：将函数执行结果返回给 LLM
5. **生成回答**：LLM 基于函数结果生成最终回答

### 5.2 函数调用的格式

```json
{
  "name": "get_weather",
  "description": "获取指定城市的天气",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "城市名称"
      }
    },
    "required": ["city"]
  }
}
```

### 5.3 函数调用的应用场景

- **检索信息**：天气、股票、百科
- **执行操作**：发送邮件、创建任务
- **数据处理**：查询数据库、调用 API
- **工具使用**：计算器、代码解释器

## 6. 多模态

**多模态（Multimodal）** 指 AI 系统能够处理多种类型的数据，如文本、图像、音频、视频等。

### 6.1 常见的多模态能力

| 模态 | 能力 | 示例 |
|------|------|------|
| **文本 + 图像** | 图像理解、图文生成 | 看图问答、根据描述生成图片 |
| **文本 + 音频** | 语音识别、语音合成 | 语音输入、文字转语音 |
| **文本 + 视频** | 视频理解 | 视频内容问答、视频摘要 |

### 6.2 多模态 LLM

| 模型 | 特点 |
|------|------|
| GPT-4o/GPT-4V | 支持图像输入 |
| Claude 3 | 支持图像理解 |
| Gemini | 支持多种模态 |
| Qwen-VL | 支持多模态 |

## 7. 技术之间的关系

```
用户提问
   ↓
[提示词工程] - 设计好的提示词
   ↓
[向量化] - 将查询转换为向量
   ↓
[向量数据库] - 检索相关文档
   ↓
[RAG] - 结合检索结果
   ↓
[函数调用] - 如需要，调用工具
   ↓
[LLM] - 生成最终回答
```

## 8. 总结

本章介绍了 AI Agent 的核心技术：

- **向量嵌入**：将文本转换为语义向量
- **向量数据库**：高效存储和检索向量
- **RAG**：结合检索和生成，增强 LLM 能力
- **提示词工程**：设计有效提示词引导 LLM
- **函数调用**：让 LLM 能够使用外部工具
- **多模态**：处理多种数据类型

这些技术是构建 AI Agent 的基础。在后续章节中，我们将深入学习这些技术的具体应用和最佳实践。
