# 一、Agent 概念入门

**智能体（Agent）** 是能够感知环境、做出决策并采取行动的自主系统。本章将介绍 Agent 的基本概念、核心组件和工作原理。

## 1. 什么是 Agent

在 AI 领域，Agent 通常指由 LLM 驱动的智能系统，能够自主执行复杂任务。

### 1.1 Agent 的核心特征

| 特征 | 描述 |
|------|------|
| **自主性（Autonomy）** | 能够在没有人类干预的情况下运行 |
| **反应性（Reactivity）** | 能够感知环境并做出响应 |
| **主动性（Pro-activity）** | 能够主动设定和追求目标 |
| **社交能力（Social Ability）** | 能够与其他 Agent 或人类交互 |

### 1.2 Agent 与传统程序的区别

| 传统程序 | Agent |
|---------|-------|
| 固定逻辑，按步骤执行 | 灵活决策，自主推理 |
| 输入 → 固定输出 | 输入 → 推理 → 行动 → 反馈 |
| 不能使用外部工具 | 可以调用工具和服务 |
| 不具备记忆能力 | 有记忆系统，记住历史 |

### 1.3 Agent 的类比

可以把 Agent 想象成一个"数字员工"：
- 有大脑（LLM）
- 有记忆（短期和长期）
- 有工具（外部服务）
- 有目标（任务）

## 2. Agent 的核心组件

一个完整的 Agent 通常包含以下核心组件：

```
┌─────────────────────────────────────────┐
│           Agent                         │
│  ┌─────────────────────────────────┐  │
│  │      LLM（大脑）                │  │
│  │    - 推理                       │  │
│  │    - 决策                       │  │
│  │    - 规划                       │  │
│  └─────────────────────────────────┘  │
│  ┌─────────────────────────────────┐  │
│  │      记忆系统                   │  │
│  │    - 短期记忆（对话历史）       │  │
│  │    - 长期记忆（知识库）         │  │
│  └─────────────────────────────────┘  │
│  ┌─────────────────────────────────┐  │
│  │      工具系统                   │  │
│  │    - 工具调用                   │  │
│  │    - 工具注册                   │  │
│  │    - 结果处理                   │  │
│  └─────────────────────────────────┘  │
│  ┌─────────────────────────────────┐  │
│  │      规划与推理                 │  │
│  │    - 任务分解                   │  │
│  │    - 步骤规划                   │  │
│  │    - 反思与优化                 │  │
│  └─────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 2.1 LLM（大脑）

LLM 是 Agent 的核心，负责：
- 理解用户意图
- 推理和决策
- 生成响应
- 规划步骤

### 2.2 记忆系统

记忆系统帮助 Agent 记住历史信息：

| 记忆类型 | 描述 | 示例 |
|---------|------|------|
| **短期记忆** | 当前对话的上下文 | 用户的历史提问、之前的工具调用 |
| **长期记忆** | 持久化的知识库 | 用户偏好、文档知识库、RAG 系统 |

### 2.3 工具系统

工具系统让 Agent 能够与外部世界交互：
- 函数调用
- API 访问
- 数据库查询
- 网页搜索
- 代码执行

### 2.4 规划与推理

规划与推理让 Agent 能够解决复杂问题：
- 任务分解
- 多步规划
- 反思与优化
- 错误恢复

## 3. Agent 的工作流程

### 3.1 基本工作循环

```
用户输入
   ↓
[感知] - 理解用户意图
   ↓
[推理] - 思考如何响应
   ↓
[规划] - 决定行动计划
   ↓
[行动] - 调用工具或直接回答
   ↓
[观察] - 观察行动结果
   ↓
[反思] - 评估结果，调整策略
   ↓
循环（直到任务完成）
```

### 3.2 ReAct 模式

ReAct（Reasoning + Acting）是一种常见的 Agent 模式：

```
思考：我需要先查询天气，然后给出建议
行动：调用 get_weather(城市="北京")
观察：北京今天晴天，温度 20°C
思考：现在我知道天气了，可以给出建议
行动：给出最终回答
```

## 6. 三大工程与 Agent 的关系

在后续章节中，我们将深入学习的三大工程都是为了构建更好的 Agent：

| 工程 | 作用于 | 目标 |

## 7. 构建第一个简单 Agent

让我们看一个简单的 Agent 示例：

```python
class SimpleAgent:
    def __init__(self):
        self.memory = []  # 短期记忆
    
    def process(self, user_input):
        # 1. 添加到记忆
        self.memory.append({"role": "user", "content": user_input})
        
        # 2. 构建提示词
        prompt = self._build_prompt(user_input)
        
        # 3. 调用 LLM
        response = self._call_llm(prompt)
        
        # 4. 添加到记忆
        self.memory.append({"role": "assistant", "content": response})
        
        return response
    
    def _build_prompt(self, user_input):
        return f"""
你是一个友好的助手。
对话历史：{self.memory}
用户问题：{user_input}
请回答用户的问题。
"""
    
    def _call_llm(self, prompt):
        # 这里调用实际的 LLM API
        return "这是我的回答"
```

## 8. Agent 的挑战

构建好的 Agent 面临一些挑战：

- **提示词设计**：好的提示词是成功的关键
- **上下文管理**：有效利用有限的上下文窗口
- **工具使用**：正确选择和使用工具
- **错误处理**：优雅处理失败情况
- **可观测性**：调试和监控 Agent 行为
- **安全性**：防止滥用和安全问题

## 9. 总结

本章介绍了 Agent 的基本概念：

- **Agent 定义**：自主的智能系统
- **核心组件**：LLM、记忆、工具、规划
- **工作流程**：感知 → 推理 → 规划 → 行动 → 观察 → 反思
- **应用场景**：个人助理、代码助手、数据分析等
- **三大工程**：提示词工程、上下文工程、Harness 工程
