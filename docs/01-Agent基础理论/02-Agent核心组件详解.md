# 一、Agent 核心组件详解

本章深入探讨 Agent 的核心组件：大脑、记忆、规划、行动、工具。

## 1. 核心组件架构

### 1.1 整体架构

一个完整的 AI Agent 系统通常包含以下核心组件：

```
┌─────────────────────────────────────────────────────────┐
│                     Agent 系统                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────────┐    ┌──────────────┐    ┌───────────┐  │
│  │   大脑       │───▶│   规划       │───▶│   行动    │  │
│  │  (LLM)      │    └──────────────┘    └───────────┘  │
│  └──────────────┘          │                   │       │
│         │                  │                   ▼       │
│         │                  │            ┌───────────┐  │
│         ▼                  │            │   工具    │  │
│  ┌──────────────┐         │            └───────────┘  │
│  │    记忆      │◀────────┘                           │
│  └──────────────┘                                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 1.2 组件职责

| 组件 | 职责 | 关键能力 |
|------|------|---------|
| **大脑** | 决策与推理 | 理解、思考、决策 |
| **记忆** | 信息存储与检索 | 短期记忆、长期记忆 |
| **规划** | 任务分解与执行计划 | 分解任务、制定计划 |
| **行动** | 执行具体操作 | 工具调用、信息输出 |
| **工具** | 扩展能力 | 外部 API、数据库等 |

---

## 2. 大脑（LLM）

### 2.1 核心地位

**大脑** 是 Agent 的核心，通常由大语言模型（LLM）担任。

**职责：**
- 理解用户意图和问题
- 推理与决策
- 生成响应内容
- 协调其他组件

### 2.2 选择考量

| 因素 | 说明 | 示例 |
|------|------|-----|
| **能力** | 理解、推理、生成能力 | GPT-4o, Claude 3.5 Sonnet |
| **成本** | API 调用费用 | GPT-3.5 更便宜 |
| **速度** | 响应延迟 | 小型模型更快 |
| **上下文窗口** | 能处理的文本量 | GPT-4o: 128K |

### 2.3 配置建议

在项目中，建议支持多模型：

```
用户选择 → 模型切换 → 不同场景用不同模型
          ├─ 简单任务: 快速/便宜
          ├─ 复杂任务: 能力强
          └─ 特定场景: 专用模型
```

## 3. 记忆系统

### 3.1 记忆分类

| 类型 | 定义 | 容量 | 用途 | 示例 |
|------|------|------|------|-----|
| **短期记忆** | 临时上下文 | 中等 | 当前对话历史 | 当前聊天记录 |
| **长期记忆** | 持久化知识 | 大 | 知识库、用户档案 | 向量数据库 |
| **工作记忆** | 正在处理的信息 | 小 | 中间状态、计划 | 任务进度 |

### 3.2 短期记忆

用于存储当前对话的上下文。

**常见实现：**
- 列表/数组存储
- 保留最近 N 轮对话
- 定期截断超过 Token 限制的部分

**数据结构：**
```
[
  {"role": "user", "content": "你好"},
  {"role": "assistant", "content": "你好，有什么可以帮到你？"},
  {"role": "user", "content": "今天天气怎样？"},
  {"role": "assistant", "content": "..."},
  ...
]
```

### 3.3 长期记忆

用于存储持久化的知识和经验。

**常见实现：**
- 向量数据库（Chroma, Pinecone, Weaviate）
- 关系型数据库
- 文件系统

**常见用法：**
- 知识库：项目文档、FAQ
- 用户档案：用户偏好、历史
- 经验库：成功的案例、失败的教训

### 3.4 在当前项目中的应用

建议采用分层记忆结构：

```
第一层（短期）：最近对话（在上下文中）
第二层（中期）：相关历史（RAG 检索）
第三层（长期）：知识库（向量化存储）
```

## 4. 规划能力

### 4.1 为什么需要规划

复杂任务无法一步完成，需要：
- 任务分解
- 分步执行
- 动态调整

### 4.2 常见规划模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **简单链** | 固定步骤顺序 | 线性任务 |
| **分支** | 根据条件选择路径 | 需要判断的任务 |
| **循环** | 重复执行直到完成 | 迭代优化任务 |
| **多 Agent** | 多个 Agent 协作 | 复杂任务 |

### 4.3 任务分解方法

**示例：写一篇技术文章**

```
原任务：写一篇关于 Agent 的文章
  ├─ 子任务 1：收集资料
  │    ├─ 查找概念定义
  │    └─ 收集案例
  ├─ 子任务 2：拟定大纲
  │    ├─ 确定主题
  │    └─ 组织结构
  ├─ 子任务 3：撰写内容
  │    ├─ 写引言
  │    └─ 写正文
  └─ 子任务 4：审核修改
       ├─ 检查逻辑
       └─ 优化表达
```

## 5. 行动与工具

### 5.1 行动类型

| 类型 | 描述 | 示例 |
|------|------|-----|
| **信息输出** | 直接回复用户 | 回答问题 |
| **工具调用** | 调用外部工具 | 查询数据库、API |
| **追问用户** | 向用户确认信息 | "你是指这个吗？" |
| **内部状态更新** | 更新记忆或进度 | 保存到数据库 |

### 5.2 工具使用

**工具定义：** 扩展 Agent 能力的外部接口。

**常见工具：**
- 📚 知识检索：查询文档
- 🔍 网络搜索：搜索信息
- 💾 数据库：查询/更新数据
- 📧 邮件发送：发送通知
- 📊 计算：执行数学计算
- 🎨 画图：生成图像
- 🔧 其他：自定义 API

**工具描述示例：**

```
工具：查询天气
描述：查询指定城市的天气
参数：城市名称
返回：天气信息（温度、湿度、风向等）
```

### 5.3 工具调用流程

```
1. 用户问题
   ↓
2. 理解问题，决定需要什么工具
   ↓
3. 准备工具参数
   ↓
4. 调用工具
   ↓
5. 获取工具返回结果
   ↓
6. 根据结果生成最终回答
```

## 6. 组件协作流程

### 6.1 完整工作流

让我们看一个完整的 Agent 工作流示例：

```
用户：我想查一下明天北京的天气，然后帮我安排一下周末的出行计划。

  ↓ [大脑]

  理解问题：
  1. 首先需要查天气
  2. 然后根据天气安排出行

  ↓ [规划]

  制定计划：
  步骤 1：调用天气查询工具
  步骤 2：根据天气结果，制定出行计划

  ↓ [行动-工具调用]

  调用工具：查询天气(城市="北京", 日期="明天")
  获取结果：晴，18-25℃，适合户外活动

  ↓ [大脑]

  基于天气结果，生成建议

  ↓ [记忆]

  保存对话历史

  ↓ [输出]

  回答用户：
    "明天北京天气很好，晴，18-25℃，推荐你可以去公园散步或者户外野餐。"
```
