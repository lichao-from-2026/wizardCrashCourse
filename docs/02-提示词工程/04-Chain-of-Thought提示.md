# 一、Chain-of-Thought 提示

Chain-of-Thought（思维链）提示是指引导 LLM 展示其推理过程，而不是直接给出答案。

## 1. 什么是 Chain-of-Thought

### 1.1 基本概念

| 类型 | 说明 | 示例 |
|------|------|------|
| **直接回答** | 直接给出答案 | 答案：42 |
| **CoT** | 展示推理过程 | 让我思考一下... 步骤1... 步骤2... 所以答案是42 |

### 1.2 CoT 的优势

相比直接回答，CoT 有以下优势：

- ✅ 提高复杂推理任务的准确率
- ✅ 让推理过程可解释
- ✅ 更容易发现错误
- ✅ 更符合人类思维方式
- ✅ 便于调试和优化

**对比示例：**

**直接回答：**
```
问题：一个游泳池，单独用A管注水需要6小时，单独用B管注水需要4小时。
如果同时用两个管注水，需要多少小时注满？

答案：2.4小时
```

**CoT：**
```
问题：一个游泳池，单独用A管注水需要6小时，单独用B管注水需要4小时。
如果同时用两个管注水，需要多少小时注满？

让我一步步思考：

1. 首先，计算每小时的注水量
2. A管每小时可以注满游泳池的 1/6
3. B管每小时可以注满游泳池的 1/4
4. 两个管一起每小时可以注满：1/6 + 1/4 = (2/12 + 3/12) = 5/12
5. 所以注满整个游泳池需要的时间是：1 / (5/12) = 12/5 = 2.4小时

答案：2.4小时
```

## 2. Zero-shot CoT

### 2.1 什么是 Zero-shot CoT

Zero-shot CoT 是指不给示例，只加上一句"让我们一步步思考"或类似的提示。

**示例：**
```
问题：小明有5个苹果，给了小红2个，又买了3个，现在小明有几个苹果？

让我们一步步思考：
```

### 2.2 常用的 Zero-shot CoT 提示

| 提示语 | 适用场景 |
|--------|---------|
| "让我们一步步思考" | 通用 |
| "请先思考，再回答" | 通用 |
| "请展示你的推理过程" | 通用 |
| "让我们仔细分析一下" | 分析类任务 |
| "让我们一步步解决这个问题" | 问题解决 |

## 3. Few-shot CoT

### 3.1 什么是 Few-shot CoT

Few-shot CoT 是指给几个带有推理过程的示例。

**示例：**
```
问题1：小明有10个糖果，吃了3个，又买了5个，现在有几个？

推理过程：
1. 初始有10个
2. 吃了3个，剩下 10 - 3 = 7 个
3. 又买了5个，现在有 7 + 5 = 12 个
答案：12个

问题2：小红有20元，买书花了8元，买笔花了5元，还剩多少钱？

推理过程：
1. 初始有20元
2. 买书花了8元，剩下 20 - 8 = 12 元
3. 买笔花了5元，剩下 12 - 5 = 7 元
答案：7元

问题3：小李有15个气球，给了朋友6个，自己又吹了4个，现在有几个？

推理过程：
```

### 3.2 示例设计原则

- 每个示例都要有完整的推理步骤
- 步骤要清晰、有条理
- 使用相同的格式
- 覆盖不同的情况

## 4. CoT 提示模板

### 4.1 基础模板

```
问题：[问题描述]

请按以下步骤回答：

步骤1：[第一步]
步骤2：[第二步]
步骤3：[第三步]
...

答案：[最终答案]
```

### 4.2 带验证的模板

```
问题：[问题描述]

请完成以下步骤：

1. 理解问题
   - 问题的关键点是什么？
   - 需要什么信息？

2. 制定计划
   - 解决问题的步骤是什么？

3. 执行计划
   - 一步步计算或推理

4. 验证答案
   - 检查是否正确
   - 有没有遗漏？

答案：
```

### 4.3 多选项模板

```
问题：[问题]

选项：
A. [选项A]
B. [选项B]
C. [选项C]
D. [选项D]

请思考：
1. 先分析每个选项
2. 排除错误选项
3. 确定正确选项

我的思考：
- 选项A：...
- 选项B：...
- 选项C：...
- 选项D：...

答案：[选项]
```

## 5. 实际应用示例

### 5.1 数学推理

```
问题：一个长方形的长是12厘米，宽是8厘米，周长是多少厘米？

请展示你的思考过程：

思考：
1. 首先回忆长方形周长的公式
2. 周长 = 2 × (长 + 宽)
3. 代入数值：长=12，宽=8
4. 计算：2 × (12 + 8) = 2 × 20 = 40
5. 所以周长是40厘米

答案：40厘米
```

### 5.2 逻辑推理

```
问题：A、B、C三个人，一个是老师，一个是医生，一个是工程师。
已知：
1. A不是医生
2. B不是老师
3. C是工程师

请问：A、B、C分别是什么职业？

让我们一步步分析：

推理：
1. 已知C是工程师
2. A不是医生，那么A只能是老师（因为C已经是工程师了）
3. B不是老师，A是老师，C是工程师，所以B只能是医生
4. 验证：
   - A：老师
   - B：医生
   - C：工程师

答案：A是老师，B是医生，C是工程师
```

### 5.3 代码调试

问题：这段代码有什么问题？

代码：
```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)

print(calculate_average([]))
```

请分析：

思考过程：
1. 看函数的功能是计算平均值
2. 如果传入空列表 []，len(numbers) 是 0
3. 会出现除以 0 的错误 ZeroDivisionError
4. 应该先检查列表是否为空

问题：除以零错误

修复方案：
```python
def calculate_average(numbers):
    if not numbers:
        return 0  # 或者抛出异常
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)
```


## 6. 在当前项目中的应用

### 6.1 在 Agent 中使用 CoT

在你的 AI Agent 项目中，CoT 可以帮助 Agent 更好地进行推理和决策。

**示例：**
```
系统提示词：
你是一个聪明的助手，需要解决复杂问题。
请总是先展示你的思考过程，然后再给出答案。
你的思考过程应该清晰、有条理。

处理步骤：
1. 理解用户的问题
2. 分析需要什么信息
3. 决定使用什么工具
4. 执行并观察结果
5. 总结答案
```

### 6.2 调试和分析

CoT 可以帮助你调试 Agent 的行为。

**好处：**
- 可以看到 Agent 的思考过程
- 容易发现哪里出错了
- 便于优化提示词

### 6.3 最佳实践

在项目中使用 CoT 的最佳实践：

| 实践 | 说明 |
|------|------|
| **任务选择** | CoT 对复杂推理任务最有效 |
| **提示位置** | 思考提示放在问题之后，答案之前 |
| **长度控制** | 思考过程不要太长，保持清晰 |
| **灵活运用** | 根据任务类型调整 CoT 的方式 |

### 6.4 注意事项

- CoT 会增加输出长度，注意上下文窗口
- 对简单任务可能不需要 CoT
- 可以根据任务复杂度选择是否使用
- 注意 CoT 可能增加成本

## 7. 总结

Chain-of-Thought 提示要点：
- ✅ 引导 LLM 展示推理过程
- ✅ 提升复杂任务的准确率
- ✅ Zero-shot CoT 和 Few-shot CoT
- ✅ 在 Agent 中使用 CoT 帮助调试和优化
- ✅ 注意平衡效果和成本
