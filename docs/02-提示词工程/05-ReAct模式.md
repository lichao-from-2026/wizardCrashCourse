# 一、ReAct 模式

ReAct（Reasoning + Acting）是指将推理和行动结合，让 LLM 像人类一样思考、行动、观察、再思考。

## 1. 什么是 ReAct

### 1.1 基本概念

| 模式 | 说明 |
|------|------|
| **只推理** | 只思考，不行动 |
| **只行动** | 直接行动，不思考 |
| **ReAct** | 思考 → 行动 → 观察 → 思考 → ... |

### 1.2 ReAct 的循环

```
思考 (Thought) → 行动 (Action) → 观察 (Observation) → 思考 (Thought) → ...
```

### 1.3 示例：

```
问题：搜索北京今天的天气怎么样？

思考：我需要查询北京今天的天气，我应该使用搜索工具。
行动：搜索"北京今天天气"
观察：北京今天多云，气温15-22度，东北风3级
思考：根据搜索结果，我现在知道了北京今天的天气。
回答：北京今天多云，气温15-22度，东北风3级。
```

## 2. ReAct 提示模板

### 2.1 标准模板

```
问题：[问题描述]

你可以使用以下工具：
- 工具1：[工具描述]
- 工具2：[工具描述]

请按以下格式回答：

思考：[你的思考]
行动：[选择的工具和参数]
观察：[工具返回的结果]
...（可以有多个思考-行动-观察循环）
思考：[总结
回答：[最终答案]
```

### 2.2 完整模板

```
系统提示词：
你是一个智能助手，可以使用工具解决问题。
请总是先思考，再行动，再观察，循环往复，直到得到答案。

可用工具：
1. 搜索：搜索网络信息
2. 计算：执行数学计算
3. 查询：查询数据库

回答：总结答案

让我们开始：

问题：[问题]

思考：
行动：
观察：
思考：
回答：
```

## 3. 实际应用示例

### 3.1 信息查询

```
问题：中国的首都是哪里？人口有多少？

思考：我需要先查中国的首都是哪里，然后再查人口。
行动：搜索"中国首都"
观察：中国的首都是北京
思考：好的，现在我需要查北京的人口。
行动：搜索"北京人口"
观察：北京常住人口约2154万人
思考：现在我有了需要的信息了。
回答：中国的首都是北京，北京常住人口约2154万人。
```

### 3.2 数学计算

```
问题：计算 1234 × 5678 的结果是多少？

思考：这是一个大数字乘法，我应该使用计算器。
行动：计算"1234 * 5678"
观察：1234 × 5678 = 7006652
思考：结果已经算出来了。
回答：1234 × 5678 = 7006652
```

### 3.3 多步推理

```
问题：小明比小红大3岁，小红比小李大5岁，小李今年10岁，问小明今年多少岁？

思考：我需要一步步计算：
1. 先算小红的年龄
2. 再算小明的年龄

思考：小李10岁，小红比小李大5岁，所以小红是 10 + 5 = 15岁
行动：计算"10 + 5"
观察：15
思考：小红15岁，小明比小红大3岁，所以小明是 15 + 3 = 18岁
行动：计算"15 + 3"
观察：18
思考：算出来了，小明18岁
回答：小明今年18岁
```

## 4. 在当前项目中的应用

### 4.1 Agent 设计

在你的 AI Agent 项目中，ReAct 是实现工具使用的核心模式。

**Agent 系统提示词：**
```
你是一个智能助手，可以使用工具解决问题。

可用工具：
1. 搜索：搜索网络信息
2. 查询：查询知识库
3. 计算：执行数学计算
4. 执行：执行系统命令

请按以下格式：

思考：[思考]
行动：[工具名] [参数]
观察：[结果]
...重复直到找到答案
回答：[答案]
```

### 4.2 工具选择策略

| 策略 | 说明 |
|------|
| **明确需求 |
| **相似场景 |
| 工具选择 |
| 需要外部信息 | 搜索、查询 |
| 需要计算 | 计算 |
| 需要执行操作 | 执行 |

### 4.3 最佳实践

在项目中使用 ReAct 的最佳实践：

| 实践 | 说明 |
|------|
| **清晰描述 | 思考过程要清晰 |
| **限制步数 | 限制最大循环步数 |
| **错误处理 | 处理工具调用失败的情况 |
| **超时控制 | 避免无限循环 |

### 4.4 注意事项

- 避免无限循环，设置最大步数
- 处理工具调用失败的情况
- 验证观察结果的准确性
- 及时总结最终答案

## 5. 总结

ReAct 模式要点：
- ✅ 思考 → 行动 → 观察 → 思考循环
- ✅ 将推理和行动结合
- ✅ 是 Agent 使用工具的核心模式
- ✅ 注意处理错误和避免循环
