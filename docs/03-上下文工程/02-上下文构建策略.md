# 一、上下文构建策略

如何构建有效的上下文，让 LLM 更好地理解和回答问题。

## 1. 上下文的组成

### 1.1 完整上下文的构成

```
完整上下文 = 
  系统提示词 + 
  相关知识 + 
  历史对话 + 
  当前问题
```

### 1.2 各部分作用

| 部分 | 作用 |
|------|------|
| **系统提示词** | 设定角色和目标 |
| **相关知识** | 提供背景信息 |
| **历史对话** | 保持对话一致性 |
| **当前问题** | 明确用户需求 |

## 2. 上下文构建原则

### 2.1 相关性原则

只包含相关的信息，不要添加无关的内容。

**示例：**
```
问题：如何学习 Python？

好的上下文：Python 学习资源、学习路线
不好的上下文：如何做菜、最近的新闻
```

### 2.2 充分性原则

提供足够的信息，让 LLM 能够全面理解。

**示例：**
```
问题：这个函数有什么问题？

好的上下文：完整的函数代码 + 使用示例 + 错误信息
不好的上下文：只有函数名
```

### 2.3 组织性原则

信息要有清晰的结构，便于理解。

**示例：**
```
1. 项目背景
2. 需求说明
3. 问题描述
4. 已尝试的方案
```

## 3. 在当前项目中的应用

### 3.1 上下文模板

在你的项目中，可以建立标准的上下文模板。

**模板示例：**
```
系统：{system_prompt}

相关信息：
{relevant_information}

历史对话：
{history}

用户问题：{question}
```

### 3.2 最佳实践

| 实践 | 说明 |
|------|------|
| **相关筛选** | 只选择真正相关的信息 |
| **重要前置** | 重要信息放在前面 |
| **结构清晰** | 使用标题和列表组织 |
| **更新及时** | 及时更新过期的信息 |

## 4. 总结

上下文构建策略要点：
- ✅ 理解上下文的组成部分
- ✅ 遵循相关、充分、组织原则
- ✅ 使用模板标准化流程
- ✅ 根据项目特点调整
