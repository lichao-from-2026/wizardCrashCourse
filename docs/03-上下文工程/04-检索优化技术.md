# 一、检索优化技术

优化 RAG 检索质量，提升相关度和准确性。

## 1. 相似度计算

### 1.1 常用相似度指标

| 指标 | 说明 |
|------|------|
| **余弦相似度** | 最常用，衡量向量角度 |
| **点积** | 向量乘积之和 |
| **欧氏距离** | 向量间距离 |

### 1.2 选择建议

- 大多数情况：余弦相似度
- 向量已归一化：点积等价于余弦相似度

## 2. 重排序（Rerank）

### 2.1 什么是重排序

**流程：**
```
1. 向量检索 → 返回 top_k 个结果（如 20 个）
2. 使用更强的模型重新排序 → 返回 top_n（如 5 个）
```

### 2.2 常用的重排序模型

- Cross-Encoder（如 BGE-Reranker）
- Cohere Rerank
- 各种专门的重排序模型

### 2.3 为什么有效

- 向量检索是粗排，速度快
- 重排序是精排，质量高
- 两者结合：速度 + 质量

## 3. 混合检索

### 3.1 混合检索策略

结合多种检索方法：

| 方法 | 说明 |
|------|------|
| **向量检索** | 语义相似度 |
| **关键词检索** | 关键词匹配（BM25） |
| **混合** | 两者结合 |

### 3.2 实现方式

```
1. 分别用向量检索和关键词检索
2. 合并结果
3. 去重
4. 重排序
```

## 4. 其他优化技术

### 4.1 调整 top_k

**选择建议：**
- 先召回较多（如 20-50）
- 然后重排序选出较少（如 5-10）

### 4.2 阈值过滤

设置相似度阈值，过滤掉不相关的结果。

### 4.3 查询扩展

改写或扩展用户查询，提高召回率。

## 5. 在当前项目中的应用

### 5.1 优化路线图

```
1. 基础向量检索
2. 调整 top_k 和分块策略
3. 增加重排序
4. 尝试混合检索
5. 查询扩展
6. 持续评估和优化
```

### 5.2 评估方法

建立评估数据集，衡量检索质量。

**指标：**
- Recall（召回率）
- Precision（精确率）
- NDCG（排序质量）

## 6. 总结

检索优化技术要点：
- ✅ 理解相似度计算
- ✅ 使用重排序提升质量
- ✅ 考虑混合检索策略
- ✅ 调整各种参数
- ✅ 持续评估和优化
