# 一、上下文压缩与优化

压缩和优化检索到的上下文，节省 Token 并提升质量。

## 1. 为什么需要压缩

### 1.1 问题

- 检索到的内容可能有冗余
- Token 数量有限，需要节省
- 不相关的内容会干扰 LLM

### 1.2 目标

- 保留关键信息
- 去除冗余内容
- 节省 Token
- 提升质量

## 2. 压缩方法

### 2.1 抽取式摘要

**方法：** 从原文中抽取关键句子。

**优点：**
- 信息准确
- 保留原文表述

**缺点：**
- 可能不够简洁
- 不够连贯

### 2.2 抽象式摘要

**方法：** 用 LLM 生成摘要。

**优点：**
- 更简洁
- 更连贯

**缺点：**
- 可能丢失细节
- 增加成本

### 2.3 关键信息提取

**方法：** 只提取关键信息，如：
- 关键概念
- 数字和日期
- 步骤和流程

## 3. 优化策略

### 3.1 去重

去除重复或相似的内容。

### 3.2 精简表达

用更简洁的方式表达相同的意思。

### 3.3 过滤不相关

过滤掉与问题关系不大的内容。

## 4. 在当前项目中的应用

### 4.1 压缩流程

```
1. 检索到文档块
2. 判断是否需要压缩
3. 选择压缩方法
4. 执行压缩
5. 验证压缩结果
6. 输入给 LLM
```

### 4.2 最佳实践

| 实践 | 说明 |
|------|------|
| **灵活选择** | 根据情况选择是否压缩 |
| **验证质量** | 确保压缩不丢失重要信息 |
| **成本考虑** | 权衡压缩成本和节省的 Token |

## 5. 总结

上下文压缩与优化要点：
- ✅ 理解压缩的必要性
- ✅ 掌握各种压缩方法
- ✅ 使用优化策略
- ✅ 在项目中灵活应用
