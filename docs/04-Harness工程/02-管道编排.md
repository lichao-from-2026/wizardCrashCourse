# 一、管道编排

使用管道和 LCEL 编排多个组件和步骤，构建复杂工作流。

## 1. 什么是管道

### 1.1 基本概念

**管道（Pipeline）**：将多个步骤串联起来，形成数据处理流。

```
输入 → 步骤1 → 步骤2 → 步骤3 → 输出
```

### 1.2 LCEL (LangChain Expression Language)

**链式调用**，用 `|` 符号连接各个步骤：

```python
chain = step1 | step2 | step3
result = chain.invoke(input)
```

## 2. 常见管道模式

### 2.1 顺序链

依次执行各个步骤：

```
输入 → 提示词模板 → LLM → 输出解析 → 输出
```

### 2.2 并行链

并行执行多个任务：

```
        ┌→ 任务A →
输入 → ┤         ├→ 合并 → 输出
        └→ 任务B →
```

### 2.3 路由链

根据条件选择不同分支：

```
        ┌→ 分支A →
输入 → 条件 ┤         ├→ 输出
        └→ 分支B →
```

### 2.4 转换链

转换数据格式：

```
输入 → 格式化 → 转换 → 输出
```

## 3. 实际示例

### 3.1 简单顺序链

```python
# 伪代码
from prompt import PromptTemplate
from llm import LLM

# 1. 提示词模板
prompt = PromptTemplate("回答问题：{question}")

# 2. LLM
llm = LLM()

# 3. 输出解析
parser = OutputParser()

# 4. 构建管道
chain = prompt | llm | parser

# 5. 调用
result = chain.invoke({"question": "你好"})
```

### 3.2 LCEL 语法

```python
# LCEL 语法
chain = (
    {
        "context": retriever,
        "question": itemgetter("question")
    }
    | prompt_template
    | llm
    | output_parser
)
```
