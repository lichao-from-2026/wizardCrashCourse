# 一、可观测性与调试

监控、日志、追踪，提升系统可观测性，便于调试和分析。

## 1. 可观测性三大支柱

### 1.1 日志（Logging）

记录发生的事件。

**级别：**
- DEBUG：详细的调试信息
- INFO：一般信息
- WARNING：警告信息
- ERROR：错误信息
- CRITICAL：严重错误

### 1.2 指标（Metrics）

统计数据。

**常见指标：**
- 请求次数
- 响应时间
- 错误率
- Token 用量

### 1.3 追踪（Tracing）

记录请求经过的各个步骤。

## 2. 工具推荐

### 2.1 日志

- Python：logging、structlog
- JS：winston、pino

### 2.2 指标

- Prometheus + Grafana
- Datadog

### 2.3 追踪

- OpenTelemetry
- Jaeger
- LangSmith（专门用于 LLM）

## 3. 调试技巧

### 3.1 逐步调试

在管道的每个步骤检查输入输出。

### 3.2 记录完整信息

记录提示词、LLM 输入输出、工具调用等。

### 3.3 可视化

可视化追踪链路，快速定位问题。

## 4. 在当前项目中的应用

### 4.1 最佳实践

| 实践 | 说明 |
|------|------|
| **结构化日志** | 使用结构化格式，便于查询 |
| **关键链路追踪** | 记录关键链路 |
| **告警机制** | 异常时及时告警 |
| **监控面板** | 实时监控系统状态 |

## 5. 总结

可观测性与调试要点：
- ✅ 理解三大支柱：日志、指标、追踪
- ✅ 使用合适的工具
- ✅ 掌握调试技巧
- ✅ 在项目中建立可观测性体系
