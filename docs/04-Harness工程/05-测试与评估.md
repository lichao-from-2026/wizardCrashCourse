# 一、测试与评估

单元测试、集成测试、端到端评估、A/B 测试等。

## 1. 测试分层

### 1.1 单元测试

测试单个组件。

**重点：**
- 每个组件独立测试
- Mock 外部依赖
- 快速运行

### 1.2 集成测试

测试多个组件的配合。

**重点：**
- 组件间的接口
- 数据流转
- 错误处理

### 1.3 端到端测试

测试完整流程。

**重点：**
- 从用户输入到最终输出
- 真实场景

## 2. 评估方法

### 2.1 自动化评估

- 正确性：答案是否正确
- 相关性：答案是否相关
- 有用性：答案是否有用
- 安全性：答案是否安全

### 2.2 人工评估

让人工评估质量。

### 2.3 A/B 测试

对比不同版本的表现。

## 3. 工具推荐

- 单元测试：pytest、jest
- 端到端：LangSmith、自定义框架
- A/B 测试：自建评估平台

## 4. 在当前项目中的应用

### 4.1 最佳实践

| 实践 | 说明 |
|------|------|
| **测试覆盖** | 核心功能有测试 |
| **持续集成** | 每次变更自动测试 |
| **评估数据集** | 维护评估数据集 |
| **定期评估** | 定期评估质量 |
