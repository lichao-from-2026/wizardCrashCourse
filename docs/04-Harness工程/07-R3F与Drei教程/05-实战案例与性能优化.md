# 实战案例与性能优化

## 一、实战项目案例

### 1.1 交互式产品展示器

**项目目标**：创建一个交互式3D产品展示系统，支持模型旋转、材质切换、环境光照调整。

**核心功能**：
- 模型加载与切换
- 材质定制与预览
- 视角控制与动画
- 环境光照调整
- 热点标注与信息展示

**技术实现**：
```typescript
import { useGLTF, OrbitControls, Environment, Html } from '@react-three/drei'

function ProductViewer() {
  const { scene, nodes, materials } = useGLTF('/models/product.glb')
  const [material, setMaterial] = useState('metal')

  return (
    <>
      <OrbitControls enableDamping dampingFactor={0.05} />
      <Environment preset="studio" />
      <primitive object={scene}>
        {Object.entries(materials).map(([name, mat]) => (
          <mesh key={name} material={material === name ? mat : null} />
        ))}
      </primitive>
      <Html position={[1, 1, 0]} center>
        <div className="hotspot">
          <h3>产品特点</h3>
          <p>高精度工艺，优质材料</p>
        </div>
      </Html>
    </>
  )
}
```

function DataVisualization({ data }) {

## 二、性能优化策略

### 2.1 渲染性能优化

**几何体优化**：
- 使用简化几何体，减少顶点数量
- 合并网格，减少draw call
- 使用实例化渲染（InstancedMesh）

**材质优化**：
- 使用合适的材质类型
- 优化纹理大小和格式
- 减少复杂材质的使用

**阴影优化**：
- 使用ContactShadows替代真实阴影
- 调整阴影贴图分辨率
- 限制阴影投射对象

### 2.2 JavaScript性能优化

**状态管理**：
- 合理使用React状态
- 使用Zustand等轻量级状态库
- 避免不必要的状态更新

**渲染循环**：
- 优化useFrame回调
- 避免在useFrame中分配内存
- 使用delta时间实现与帧率无关的动画

**组件优化**：
- 使用React.memo缓存组件
- 使用useMemo缓存计算结果
- 合理使用useCallback缓存函数

### 2.3 资源管理

**资源加载**：
- 使用Preload组件预加载资源
- 实现资源的懒加载
- 优化资源大小和格式

**内存管理**：
- 及时释放Three.js资源
- 避免内存泄漏
- 监控内存使用

### 2.4 性能监测与分析

**工具使用**：
- 使用Stats组件监测性能
- 使用Chrome DevTools分析性能瓶颈
- 使用Lighthouse评估整体性能

**优化流程**：
1. 建立性能基准
2. 识别性能瓶颈
3. 实施优化措施
4. 验证优化效果
5. 持续监控

## 三、最佳实践总结

### 3.1 开发流程

1. **需求分析**：明确项目目标和功能需求
2. **技术选型**：选择合适的技术栈和组件
3. **架构设计**：设计清晰的组件结构和数据流
4. **开发实现**：按模块实现功能
5. **性能优化**：识别并解决性能问题
6. **测试验证**：确保功能正常和性能达标
7. **部署发布**：优化构建和部署流程

### 3.2 代码规范

- **命名规范**：使用清晰、一致的命名
- **代码组织**：按功能模块组织代码
- **注释规范**：添加必要的注释
- **类型安全**：使用TypeScript确保类型安全
- **代码风格**：使用ESLint和Prettier保持一致的代码风格

### 3.3 常见问题解决方案

**场景不渲染**：
- 检查相机位置和视野
- 确认光照配置正确
- 验证对象可见性

**性能下降**：
- 检查对象数量和复杂度
- 优化渲染循环逻辑
- 减少不必要的重新渲染

**内存泄漏**：
- 及时释放Three.js资源
- 清理事件监听器
- 避免循环引用
