# 一、LLM 作为 Agent 大脑

作为 Agent 的核心「大脑」，LLM 的选择和配置至关重要。

## 1. LLM 选择因素

### 1.1 关键指标

| 指标 | 说明 | 建议 |
|------|------|------|
| **能力** | 理解、推理、工具调用、代码生成 | 根据任务复杂度选择 |
| **速度** | 响应时间 | 实时交互场景优先考虑速度 |
| **成本** | API 调用费用 | 平衡成本和效果 |
| **上下文窗口** | 可处理的文本量 | 长对话、RAG 系统需大窗口 |
| **可靠性** | 稳定性、可用性 | 生产环境优先高 SLA |

### 1.2 主流模型对比

| 模型系列 | 特点 | 适用场景 |
|---------|------|---------|
| **GPT-4o / Claude 3.5** | 能力强，综合表现好 | 复杂推理、多模态 |
| **GPT-3.5 / Claude 3** | 性价比高，速度快 | 日常任务、简单工具调用 |
| **开源模型 (Llama, Mistral)** | 可私有化部署，成本低 | 数据敏感、定制化 |

## 2. LLM 配置优化

### 2.1 关键参数

| 参数 | 作用 | 建议值 |
|------|------|--------|
| **temperature** | 控制随机性 | 0.0-0.3（确定性任务）；0.7-1.0（创意任务） |
| **top_p** | 控制采样范围 | 0.9-1.0 |
| **max_tokens** | 最大输出长度 | 根据任务调整 |
| **frequency_penalty** | 减少重复 | 0.0-0.5 |

### 2.2 在项目中的最佳实践

1. **多模型策略**：简单任务用快速模型，复杂任务用强模型
2. **缓存机制**：相同或相似问题复用 LLM 输出
3. **降级方案**：主模型不可用时切换到备选模型

## 3. 总结

LLM 是 Agent 的大脑，选择合适的模型并优化配置是成功的基础！
