# 一、记忆系统设计

记忆是 Agent 的重要组件，帮助 Agent 保持上下文、学习经验。

## 1. 记忆层次

### 1.1 三层记忆架构

| 层次 | 特点 | 存储介质 | 用途 |
|------|------|---------|------|
| **短期记忆** | 会话内上下文，实时性高 | 内存 / Redis | 当前会话状态 |
| **长期记忆** | 持久化，可检索 | 向量数据库 | 历史对话、知识 |
| **工作记忆** | 任务执行过程中的中间状态 | 内存 | 多步任务跟踪 |

### 1.2 在项目中的应用

在本项目知识库中，RAG 系统就是一种长期记忆的应用！

## 2. 记忆存储与检索

### 2.1 存储方案

- **向量数据库**：Chroma、Pinecone、Weaviate
- **关系数据库**：PostgreSQL（结构化数据）
- **文件系统**：JSON、Markdown（简单场景）

### 2.2 检索策略

| 策略 | 说明 |
|------|------|
| **语义检索** | 通过向量相似度查找相关记忆 |
| **关键字检索** | 精确匹配 |
| **混合检索** | 结合语义和关键字 |
| **时间过滤** | 优先考虑近期记忆 |

## 3. 记忆优化

- **重要性评分**：给记忆打标签，优先检索高价值内容
- **压缩总结**：对历史对话进行总结，减少冗余
- **遗忘策略**：定期清理过时或低质量的记忆

## 4. 总结

记忆系统让 Agent 能够「记住」过去，提供连贯的体验！
