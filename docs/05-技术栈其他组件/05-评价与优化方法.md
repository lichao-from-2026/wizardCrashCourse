# 一、评价与优化方法

评估和优化是让 Agent 越来越好的关键！

## 1. 评价指标

### 1.1 核心指标

| 指标 | 说明 | 测量方法 |
|------|------|---------|
| **准确率** | 回答正确的比例 | 人工标注 / 自动评估 |
| **有用性** | 对用户是否有帮助 | 用户反馈 |
| **安全性** | 是否有有害内容 | 内容审核 |
| **效率** | 响应时间、成本 | 日志分析 |
| **鲁棒性** | 处理异常情况的能力 | 压力测试 |

## 2. 评估方法

### 2.1 评估类型

| 类型 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **自动评估** | LLM 自评 / 规则检查 | 速度快、成本低 | 不够准确 |
| **人工评估** | 专家标注 | 准确 | 成本高、慢 |
| **用户反馈** | 实际用户评分 | 真实 | 样本可能有偏 |

### 2.2 在项目中的应用

我们使用 `npm run build` 检查文档和链接的正确性，这就是一种简单的自动评估！

## 3. 优化方法

### 3.1 常见优化方向

1. **提示词优化**：改进 Prompt Engineering
2. **检索优化**：调整 RAG 策略
3. **模型选择**：尝试不同 LLM
4. **工具优化**：改进工具设计
5. **流程优化**：调整 Harness 编排

### 3.2 A/B 测试

对比两种方案，选择效果更好的！

## 4. 持续迭代

评估 → 发现问题 → 优化 → 再评估 → ...

## 5. 总结

没有完美的 Agent，只有不断优化的 Agent！
